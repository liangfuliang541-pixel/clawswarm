## 🔄 技能链 (Skill Chaining)

技能可以顺序链或并行分叉：

### 顺序链 (Sequential Chain)

```
任务: "找餐厅, 获取信息, 保存到 Notion"

┌─────────┐    ┌─────────┐    ┌─────────┐
│ web_    │───▶│  docx   │───▶│  MCP    │
│ search  │    │ (解析)  │    │ notion  │
└─────────┘    └─────────┘    └─────────┘
   │              │              │
   ▼              ▼              ▼
 [查询]     [结构化数据]    [保存到
                      数据库]
```

### 并行分叉 (Parallel Fork)

```
任务: "并行搜索 5 个不同主题"

                    ┌─── web_search (主题1)
                    ├─── web_search (主题2)
┌─────────┐  ──────▶├─── web_search (主题3) ──▶ 聚合器 ──▶ 报告
│  任务   │         ├─── web_search (主题4)
└─────────┘         └─── web_search (主题5)
```

## 🎯 能力匹配 (Capability Matching)

Master 根据能力将任务匹配到节点：

```python
class CapabilityMatcher:
    """将任务需求与节点能力匹配"""
    
    def match(self, task, nodes):
        # 1. 从任务中提取所需能力
        required = task.get_requirements()  # ["search", "write", "notion"]
        
        # 2. 找到具有所有所需能力的节点
        candidates = []
        for node in nodes:
            node_caps = node.get_capabilities()  # ["search", "write", "code"]
            
            if all(cap in node_caps for cap in required):
                candidates.append(node)
        
        # 3. 按偏好评分
        scored = self.score_by_preference(task, candidates)
        
        # 4. 返回最佳匹配
        return max(scored, key=lambda n: n.score)
```

## 🌐 Hub-Spoke 跨公网通信 (v0.9+)

无需 tunnel、SSH、公网 IP 的跨机器通信方案。

```
Hub (port 18080) ←──── HTTP 轮询 ──── Agent (VM/远程)
```

核心组件：`networking.py`（HubServer + HubAgent + HubClient），嵌入 `master_api.py`。

详见 [MODULES_CN.md](../MODULES_CN.md) networking.py 章节。

## 🔌 Agent 适配器架构 (v0.11+)

可插拔适配器层，支持异构 Agent 类型：

```
HubAgent.execute_task()
    │
    ├── adapter_type=None → 原生 echo
    ├── adapter_type="hermes" → HermesAdapter (ACP JSON-RPC 2.0)
    ├── adapter_type="evolver" → EvolverAdapter (sessions_send/文件轮询)
    └── adapter_type="openclaw" → OpenClawAdapter (HTTP Hub)
```

详见 [MODULES_CN.md](../MODULES_CN.md) 适配器章节。
